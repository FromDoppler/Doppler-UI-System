Bienvenido a Doppler Styles Guides

UI Styles Guides en vivo. Esta biblioteca está destinada a ser un lenguaje visual de Doppler, para centralizar todas las marcas en un solo lugar para todos nuestros productos, reutilización de código y esfuerzo.

También permite desacoplar la interfaz de usuario de la lógica de Front End y trabajar de una manera más ordenada.

¿Cómo puedo usar esta biblioteca en mi proyecto?

Simplemente incluye el archivo css y js en tu html.

Verifique la versión correcta en la pestaña de versiones de github primero y reemplace la vx.x.x siguiente.

¿Cómo aplicar componentes y usar mixins?

En nuestra biblioteca utilzamos Sass Doc para crear nuestra documentación. Consulte su uso en Sass Doc

Consulte nuestra documentación (en desarrollo)

Contribuyendo a este proyecto

Si desea contribuir a este proyecto, debe seguir las reglas:

  • Utilizamos Conventional Commits por favor, compruebe su uso aquí: Conventional Commits
  • Los Commits son cortos, por lo que pueden ser fáciles de seguir y leer.
  • Los colores, el espaciado y los tamaños de fuente se almacenan en variables para su reutilización.
  • Al hacer un PR (pull request), verifique el código de compilación del cdn generado automáticamente, mire a través del número de compilación en el enlace de verificación del generador de solicitudes de extracción de jenkins, debería ser algo como http://docker.fromdoppler.com:8080/job/doppler-ui-system/job/pull-requests/391/ en este ejemplo 391 si construye, su enlace será http://cdn.fromdoppler.com/doppler-ui-library/build391/index.html usted puede usar este enlace para también incluya y pruebe en su página web, apuntando a css y js, como se explicó anteriormente.
  • Considere obtener la aprobación de al menos uno de nuestros colaboradores habituales.
  • Tenga en cuenta que esta biblioteca puede tener muchos clientes, pensamos en la retrocompatibilidad antes de realizar cambios.
  • Haga un Fork de este proyecto para comenzar a contribuir.

¿Cómo trabajar con el código?

Una vez que haya descargado:

  • npm install instalar dependencias.
  • gulp run para ejecutar el servidor de desarrollo y la vista previa.
  • npm run prettier para formatear nuestro código, nos ahorra tiempo y energía.
  • npm run prettier-check para asegurarnos de que nuestro código este correcto.
  • gulp dist para la tarea de generación (Implementar).
  • gulp build para definir tarea de construcción.
  • gulp doc para crear documentación

Este proyecto utiliza Sass, SassDocs, javascript y JQuery. En este momento estamos basando todos nuestros diseños en Flexbox.
